1. What is the difference between invoking and creating a function?

    2. When you create a function, you are describing what it should do, but you are not running it yet. When you invoke a function, you are executing the code inside it.

  1. Explain the syntax regarding semicolons and code blocks.

    2. Semicolons (;) are used to mark the end of a statement, telling the program that an instruction is complete, while code blocks ({ }) group multiple statements together so they are treated as a single unit, such as the body of a function, loop, or conditional. In JavaScript, semicolons are technically optional due to automatic semicolon insertion, but using them consistently helps prevent errors. Code blocks are required when more than one statement must run together and are defined by curly braces.

  1. What happens when we try to store values from a function when it doesn't return any?

    2. When a function does not return a value, it implicitly returns undefined in JavaScript. So if you try to store its “result” in a variable, that variable will hold undefined.

  1. How do we separate parameters from each other when creating a function?

    2. When creating a function, parameters are separated by commas inside the parentheses () that follow the function name.

  1. Explain how parameter names are related during the invocation and creation of a function.

    2. When you create a function, you give names to its parameters—these are like placeholders for values that the function will use later. When you invoke (call) the function, you provide arguments, which are the actual values. The names of the parameters in the function definition do not have to match the variable names you pass in during invocation; what matters is the order of the arguments.

  1. Explain why a parameter is undefined when a function call is missing a parameter.

    2. A parameter becomes undefined when a function is called without providing a value for it because, in JavaScript, all parameters are optional by default. If an argument is missing, the function still creates the parameter as a local variable, but since no value was passed, its value is automatically set to undefined.

  1. What is the difference between invoking and creating a function?

    2. When we use a recursive function, the function calls itself to solve a smaller or simpler version of the same problem. Each call creates a new execution context (a new “instance” of the function) on the call stack, remembering its own parameters and state. This continues until a base case is reached, which stops further recursion. Then the function calls start returning values, and the results “unwind” back through all the previous calls.
