  6. In task 8, we will use a "callback" routine when sorting. Explain how such a "callback" routine works.

    7. A callback is a function you give to another function so it can call it whenever it needs to, like giving instructions for sorting.

    In your rNum.sort((a, b) => a - b); example:
    - The .sort() function can take a callback to decide how to compare two elements
    -The callback (a, b) => a - b is run for each pair of elements in the array.
    -If it returns a negative number, a comes before b.
    -If it returns 0, the order stays the same.
    -If it returns a positive number, b comes before a.

  7. What happens in the sorting routines you have created? Try to explain how the array can figure out if an element should come before or after another. The best way to answer this is to actually print both parameters each time the sorting routine runs

    8. hen we sort an array with a callback like (a, b) => a - b, the sort function repeatedly takes two elements from the array and calls the callback with them. The callback then decides their order: if it returns a negative number, the first element goes before the second; if it returns zero, their order stays the same; and if it returns a positive number, the second element goes before the first. The array keeps doing this for many pairs of elements until all of them are in the correct order. By printing a and b inside the callback, we can see exactly which numbers are being compared each time, which helps us understand how the array figures out the sorting order.
